# MewCode 初始对话能力 Spec

## 背景
MewCode 是一个从零开始构建的终端 AI 编程助手，目标形态类似 Claude Code。本阶段先建立最小可用的交互式对话能力：用户在终端启动 MewCode 后进入 TUI，可以输入问题，系统调用大模型 API，并把回复以流式方式输出到终端。

当前阶段只做纯对话能力，不做工具调用、文件操作、代码编辑等 Agent 能力。系统需要支持多轮对话，让模型在后续回复中能够参考之前的用户输入和助手回复。

## 目标
- 用户可以从终端启动 MewCode，并进入可持续交互的对话界面。
- 用户输入问题后，MewCode 调用配置指定的大模型后端，并以流式方式逐步显示回复。
- MewCode 支持多轮对话上下文，后续回复能够基于前文继续。
- MewCode 支持 Anthropic Claude 和 OpenAI 两类 API 后端，并能通过 YAML 配置切换。
- MewCode 的 Provider 层保持统一抽象，便于后续增加新的模型后端。
- MewCode 支持 Claude 的 extended thinking 配置。

## 功能需求
- F1: 启动后进入交互式终端对话界面，用户可以连续输入多轮消息。
- F2: 每轮用户输入后，系统读取当前配置，调用对应的大模型 API 后端生成回复。
- F3: 模型回复必须以流式方式输出，用户能看到内容逐步出现在终端，而不是等待完整结果一次性返回。
- F4: 系统维护当前会话的多轮上下文，并在后续请求中携带历史消息。
- F5: 系统支持 Anthropic Claude 协议后端。
- F6: 系统支持 OpenAI 协议后端。
- F7: 系统通过 YAML 配置文件管理 LLM 供应商信息。
- F8: YAML 配置支持六个字段：`name`、`protocol`、`model`、`base_url`、`api_key`、`thinking`。
- F9: `protocol` 字段决定使用哪一种 API 协议，至少支持 Claude 和 OpenAI 两种取值。
- F10: `thinking` 字段为可选字段，用于控制 Claude extended thinking 是否启用。
- F11: Provider 层对上层对话流程暴露统一行为，使 TUI 不需要关心具体调用的是 Claude 还是 OpenAI。
- F12: 当配置缺失、协议不支持、认证失败或 API 调用失败时，系统向用户显示可理解的错误信息，并保持程序可继续交互或可明确退出。

## 非功能需求
- N1: 使用 Python 实现。
- N2: 流式输出基于 SSE 或兼容 SSE 的流式响应机制。
- N3: Provider 抽象应避免把具体供应商协议细节泄漏到 TUI 层。
- N4: 配置文件格式应清晰、可手写、易于后续扩展多个供应商配置。
- N5: 终端交互在常见 Windows PowerShell 环境下可运行。
- N6: 用户可见文本、错误提示和新增注释使用中文。

## 不做的事
- 本阶段不实现 tool use。
- 本阶段不实现文件读取、文件写入、搜索、代码编辑或命令执行能力。
- 本阶段不实现自动修改项目代码的 Agent 工作流。
- 本阶段不实现长期记忆、会话持久化或历史记录跨进程恢复。
- 本阶段不实现多配置交互选择界面；配置切换只要求能通过配置文件完成。
- 本阶段不实现复杂主题、布局或鼠标交互，TUI 只需满足基础对话输入和流式输出。

## 验收标准
- AC1: 在终端启动 MewCode 后，用户能看到交互式输入提示，并能输入第一条消息。
- AC2: 使用 Claude 协议配置时，输入一条真实问题后，MewCode 调用 Claude 后端并流式显示回复。
- AC3: 使用 OpenAI 协议配置时，输入一条真实问题后，MewCode 调用 OpenAI 后端并流式显示回复。
- AC4: 在同一会话中连续提问时，第二轮问题可以引用第一轮内容，模型回复能体现上下文。
- AC5: 回复内容不是等完整生成后一次性打印，而是在生成过程中逐步出现在终端。
- AC6: YAML 配置中的 `name`、`protocol`、`model`、`base_url`、`api_key` 字段被正确读取并用于请求。
- AC7: YAML 配置未提供 `thinking` 字段时，系统仍能正常发起普通对话请求。
- AC8: YAML 配置启用 `thinking` 且使用 Claude 协议时，请求包含 extended thinking 所需配置。
- AC9: 当 `protocol` 是不支持的值时，系统显示明确错误，说明该协议不受支持。
- AC10: 当 API 调用失败时，系统显示可理解的失败信息，并不会输出 Python 异常堆栈作为主要用户界面。
- AC11: Provider 层可以在不修改 TUI 对话流程的前提下切换 Claude 和 OpenAI 后端。
- AC12: 端到端测试在 tmux 中启动 MewCode、输入真实对话，并能对照 checklist.md 完成验收。
